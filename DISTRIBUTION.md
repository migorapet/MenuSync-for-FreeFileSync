# Distribution Guide

## Current status

MenuSync for FreeFileSync is beta software developed on the macOS 27 beta.
Its deployment target is macOS 14, but compatibility across every macOS release
from 14 through 27 has not yet been fully tested.

MenuSync for FreeFileSync is an independent, unofficial companion. It is not
affiliated with, endorsed by, or distributed by the FreeFileSync project.

FreeFileSync is a separate dependency and is not bundled. The standard path is
`/Applications/FreeFileSync.app`, but users can select any installed
`FreeFileSync.app` in Preferences. The companion validates and launches:

```text
<selected app>/Contents/MacOS/FreeFileSync
```

## Local test build

Run:

```bash
./scripts/package_local.sh
```

Outputs:

```text
dist/MenuSync for FreeFileSync.app
dist/MenuSync-for-FreeFileSync-0.1.0-beta.zip
```

The local package is a universal `arm64 + x86_64` binary with an ad-hoc
hardened-runtime signature. It is intended for testing on the machine that
built it.

## Public beta distribution

An ad-hoc signature is not appropriate for downloads shared with other users.
For direct distribution outside the Mac App Store:

1. Join the Apple Developer Program and create a **Developer ID Application**
   certificate.
2. In Xcode, select the app target and configure your Team under **Signing &
   Capabilities**.
3. Choose **Product → Archive**.
4. In **Window → Organizer**, select the archive and choose **Distribute App**.
5. Select **Developer ID**, then **Upload** to send it to Apple's notary service.
6. Export the notarized application and test it on a separate Mac account or
   machine before publishing it.

Apple documents the certificate process in
[Developer ID certificates](https://developer.apple.com/help/account/certificates/create-developer-id-certificates/)
and the Organizer/notarization workflow in
[Notarizing macOS software before distribution](https://developer.apple.com/documentation/security/notarizing-macos-software-before-distribution).

For public releases, retain the hardened runtime, use a secure timestamp, and
do not use the ad-hoc package generated by `package_local.sh`.
